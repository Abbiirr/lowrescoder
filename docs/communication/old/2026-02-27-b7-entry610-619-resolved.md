## Root Cause Analysis — 2 Remaining Infra Fails

### scikit-learn (affects 4/24 pilot tasks: sklearn-11310, 25747, 14309, 12462)

**Error:** Cython 3.0 compilation fails on sklearn 0.20/0.21 `.pyx` files:
```
Error compiling Cython file: ... WeightedPQueueRecord ...
```
**Root cause:** `install_build_deps()` installs unconstrained `Cython` (resolves to 3.0.12). Cython 3.0 changed `nogil` syntax and other directives — incompatible with sklearn 0.20-era `.pyx` files.
**Fix:** Constrain to `Cython<3` in build deps.

### matplotlib (affects 4/24 pilot tasks: mpl-13989, 22711, 22835, 26020)

**Error:** gcc fails compiling `src/checkdep_freetype2.c`:
```
gcc ... -c src/checkdep_freetype2.c -o build/temp.../src/checkdep_freetype2.o
error: command 'gcc' failed with exit status 1
```
**Root cause:** Missing `-dev` system packages for freetype2 and libpng. `install_build_deps()` only installs `gcc g++ make git python3-dev pkg-config`. Matplotlib C extensions need `libfreetype6-dev` and `libpng-dev`.
**Fix:** Add `libfreetype6-dev libpng-dev` to apt install.

## Proposed Fix (2 options)

### Option A: Enrich `install_build_deps()` (simpler)

Add `libfreetype6-dev libpng-dev libopenblas-dev` to apt install and constrain `Cython<3`:
```python
def install_build_deps(name: str) -> subprocess.CompletedProcess[str]:
    return docker_exec(
        name,
        "apt-get update -qq && "
        "apt-get install -y -qq "
        "gcc g++ make git python3-dev pkg-config "
        "libfreetype6-dev libpng-dev libopenblas-dev && "
        "pip install -q 'Cython<3' numpy pytest",
        timeout=300,
    )
```
- Pro: 1 line change, no manifest schema change
- Con: Installs unnecessary packages for tasks that don't need them (~30s overhead)

### Option B: Manifest-driven `extra_deps` (more flexible)

Add `extra_apt_deps` and `extra_pip_deps` fields to task manifest:
```json
{"task_id": "scikit-learn__scikit-learn-11310", "extra_pip_deps": ["Cython<3"]}
{"task_id": "matplotlib__matplotlib-13989", "extra_apt_deps": ["libfreetype6-dev", "libpng-dev"]}
```
Runner installs these after `install_build_deps()` and before setup commands.
- Pro: Per-task control, no unnecessary installs
- Con: More code (manifest parsing + runner logic), schema change

**My recommendation: Option A** — simplest path to unblock all 8 affected tasks. `Cython<3` is safe for all repos (Cython 0.29.x is universally compatible). The extra apt packages add minimal overhead and don't conflict.

## B7 Agent Quality — Harness Tooling Improvements

Entry 613 confirmed B7 infrastructure is solid (0 infra fails). The problem was **agent quality** — the harness tooling crippled glm-4.7-flash in three ways. All three are now fixed.

### Problem Analysis (from B7 R7 failures)

1. **No edit_file tool** — only `write_file` (full-file replacement). The model cannot reproduce 8000-line files to change 2-3 lines. This destroyed matplotlib `_axes.py`, corrupted django `aggregates.py`, and prevented sklearn fixes entirely.
2. **Early exit on text-only response** — AgentLoop returned immediately when the model talked instead of calling tools. The sklearn agent read files 22 times but never wrote a fix because it described the fix in text and the loop ended.
3. **No self-verification** — In Docker mode, the agent was told "Do NOT run tests yourself" with no alternative. It made a fix and hoped, getting feedback only from the outer retry loop.

### Fixes Implemented (4 RCs)

**RC1: `edit_file` tool (Critical)**
- New `edit_file(path, old_string, new_string)` function in `src/autocode/utils/file_tools.py`
- Search/replace with exact-match-once semantics — errors on 0 matches, >1 matches, or empty old_string

- Fuzzy mapping updated in `src/autocode/layer4/llm.py` — `edit_file` removed from write_patterns, new `edit_patterns` set added (modify_file, patch_file, replace_in_file, search_replace, str_replace)

**RC2: Text-only nudge (Critical)**

- In AUTO mode only: when model produces text without tool calls, injects nudge message ("You must call tools to make changes. Call a tool now.") and continues the loop
- After 2 nudges, accepts the text response as before
- Non-AUTO modes (SUGGEST, READ_ONLY) unchanged — returns immediately as before

**RC3: `run_tests` tool for Docker mode (Important)**
- In `scripts/adapters/autocode_adapter.py`: when `_container_name` is set and `task.grading_command` exists, registers a `run_tests` tool
- Handler calls `docker_exec(container_name, grading_command, timeout=120)`

- `requires_approval=False` (auto-approve in benchmark mode)

**RC4: Prompt updates**
- Docker prompt Step 4: "Use edit_file to fix SOURCE code" (was "Use write_file")
- Docker prompt Step 5: "Use run_tests to verify your fix" (was "Do NOT run tests yourself")
